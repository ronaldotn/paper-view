1. Introduction to Page Insertion API

The Page Insertion API provides powerful methods for dynamically adding, removing, and manipulating pages in a PaperView document. This is essential for creating professional documents with cover pages, tables of contents, title pages, and appendices.

1.1 Overview

Traditional paged media workflows render content sequentially from start to finish. The Page Insertion API breaks this paradigm by allowing you to:

Insert pages at any position in the document, before or after the main content flow. Add blank pages for layout purposes, such as ensuring chapters start on right-hand pages. Inject custom content like covers, TOCs, and appendices. Remove or replace pages dynamically based on content changes. Re-index page numbers automatically after any modification.

1.2 Use Cases

Cover Pages - Add a professionally designed cover page before the main content.

Table of Contents - Generate a dynamic TOC that can be inserted at any position.

Title Pages - Create separate title pages for different sections or chapters.

Blank Pages - Insert blank pages to ensure chapters start on recto (right-hand) pages.

Appendices - Add supplementary content pages at the end of the document.

Dynamic Content - Insert pages based on runtime conditions or user input.

Note: This document itself demonstrates the Page Insertion API. The cover page, TOC, title page, and appendix you see were all inserted dynamically using the API methods described here.

2. API Reference

2.1 insertPage(index, options)

The core method for inserting a single page at a specific position.

const page = chunker.insertPage(0, { blank: false, content: '<h1>Cover Page</h1>', name: 'cover', className: 'my-cover', pageNumberOverride: null, breakBefore: false });
Parameter Type Description
index number Position to insert (0 = before first page)
options.blank boolean Create a blank page without content
options.content string|HTMLElement Custom content to inject into the page
options.name string Named page type for CSS @page rules
options.className string Additional CSS class for the page element
options.pageNumberOverride number Override the page counter value
options.breakBefore boolean Force a page break before insertion

2.2 insertPages(startIndex, pageConfigs)

Insert multiple pages at once with an array of configurations.

const pages = chunker.insertPages(0, [ { blank: true, name: 'cover' }, { content: tocHTML, name: 'toc' }, { blank: true, name: 'title' } ]);

2.3 prependPage(options)

Shorthand for inserting a page at the beginning (index 0).

const coverPage = chunker.prependPage({ content: '<h1>Document Title</h1>', name: 'cover' });

2.4 appendPage(options)

Insert a page at the end of the document.

const appendix = chunker.appendPage({ content: '<h2>Appendix A</h2>', name: 'appendix' });

2.5 removePage(index)

Remove a page at a specific index. Returns true if successful.

const removed = chunker.removePage(2);

2.6 replacePage(index, options)

Replace a page at a specific index with new content.

const newPage = chunker.replacePage(1, { content: '<h2>Updated Content</h2>' });

3. Events

The Page Insertion API emits events that you can listen to for custom behavior.

3.1 pageInserted

Emitted when a page is successfully inserted. Provides the page instance and its index.

chunker.on('pageInserted', (page, index) => { console.log(`Page inserted at index ${index}`); console.log('Page element:', page.element); });

3.2 pageRemoved

Emitted when a page is removed. Provides the index of the removed page.

chunker.on('pageRemoved', (index) => { console.log(`Page removed at index ${index}`); });

4. Best Practices

4.1 Insert Pages Before Rendering

For best performance, insert pages before the main content rendering begins. This avoids re-layout operations.

4.2 Use Named Pages

Use the name option to apply specific @page CSS rules to inserted pages. This allows different headers, footers, and margins for different page types.

4.3 Handle Page Numbering

Use pageNumberOverride to control page numbering. For example, set cover pages to not count towards the page number, or start numbering from a specific value.

4.4 Content Injection

When injecting HTML content, ensure it is well-formed and does not contain elements that might conflict with the page template structure.

5. Conclusion

The Page Insertion API transforms PaperView from a simple content paginator into a full-featured document layout engine. By combining page insertion with other CSS Paged Media features like leaders, page floats, footnotes, and generated content, you can create professional-quality publications entirely from web technologies.

Key takeaways: Use insertPage() for precise control over page positioning. Use prependPage() and appendPage() for common insertion patterns. Use removePage() and replacePage() for dynamic content updates. Listen to events for custom behavior and logging. Combine with named pages for different layouts per page type.
