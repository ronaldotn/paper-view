Web Workers for Layout

Offload layout calculations to background threads

How Web Workers Help

Layout calculations in paged media can be computationally intensive, especially for documents with complex content, many images, or large amounts of text. By moving these calculations to Web Workers, we keep the main thread responsive for user interactions.

The worker-based approach serializes the DOM content and sends it to background workers for analysis. Workers calculate break points, detect overflow, and determine where content should split across pages. The main thread then applies these decisions to the actual DOM.

Usage

// Enable Web Workers for layout
const previewer = new PaperView.Previewer({
    viewMode: "single",
    useWorkers: true,
    workerOptions: {
        workerCount: 2  // Number of workers (default: 2)
    }
});

// Listen for worker events
previewer.on("workersReady", (data) => {
    console.log(`${data.workerCount} workers ready`);
});

previewer.on("workerTaskComplete", (data) => {
    console.log(`Task ${data.taskId} completed in ${data.duration}ms`);
});

// Get worker stats
const stats = previewer.getWorkerStats();
console.log(stats);
// {
//   workerCount: 2,
//   readyWorkers: 2,
//   initialized: true,
//   activeTasks: 0,
//   queuedTasks: 0,
//   busyWorkers: 0,
//   idleWorkers: 2
// }

// Render as usual
const flow = await previewer.preview(content, renderTo, stylesheets);

Architecture

  1. LayoutWorkerManager: Manages a pool of Web Workers
  2. layout.worker.js: Worker script that processes layout tasks
  3. Task Queue: Tasks are queued and distributed to available workers
  4. Serialization: DOM content is serialized for worker processing
  5. Result Application: Main thread applies worker results to DOM
