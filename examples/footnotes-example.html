Understanding CSS Footnotes in Paged Media

Footnotes are an essential typographic feature for academic and technical publications. The CSS Paged Media specification defines a comprehensive model for handling footnotes, including the float: footnote property, footnote-call pseudo-elements, and footnote markers.The CSS Paged Media Module Level 3 specification defines the footnote model. See https://www.w3.org/TR/css-page-3/ for details.

The Footnote Model

When an element is assigned float: footnote, it is removed from the normal flow and placed in a footnote area at the bottom of the page. A footnote-call is inserted at the original location in the text, typically displayed as a superscript number.The footnote-call is automatically generated by the user agent and inserted inline at the point where the footnote element appeared in the source.

The footnote area is a special region at the bottom of each page that collects all footnotes referenced on that page. Footnotes are numbered sequentially within each page or throughout the document, depending on the counter configuration.Some implementations allow footnote numbering to restart on each page, while others continue numbering throughout the document. PaperView supports both approaches through the counter system.

Footnote Display Modes

The footnote-display property controls how footnotes are rendered in the footnote area:

  • block (default): Each footnote is displayed as a block element with the marker on its own line or inline with the content.
  • inline: Footnotes are displayed inline, one after another, separated by spaces or other delimiters.
  • compact: A compact display mode that minimizes vertical space between footnotes.

The choice of display mode depends on the publication style and the amount of footnote content. Academic publications typically use block display, while some technical manuals may prefer compact display to save space.The compact display mode is particularly useful when footnotes are short and numerous, such as in legal documents or technical specifications.

Footnote Policy

The footnote-policy property determines how the user agent handles footnotes when there is insufficient space on the page:

  • line (default): If the footnote reference fits on the line but the footnote area is full, the footnote may be moved to the next page.
  • keep: The footnote reference and its footnote are kept together on the same page. If there is not enough space, both are moved to the next page.
  • block: Similar to keep, but applies to block-level elements containing footnote references.

The policy setting affects the reading experience significantly. The keep policy ensures that readers can always find the footnote on the same page as the reference, but may result in more page breaks.In practice, most publications use the "line" policy as it provides a good balance between keeping references with their footnotes and avoiding excessive page breaks.

Implementation Details

PaperView's footnote implementation follows the CSS Paged Media specification closely. When a document is processed:

  1. Elements with float: footnote are identified during CSS parsing.
  2. Each footnote element is replaced in the text flow with a footnote-call span containing the footnote number.
  3. The original footnote content is collected and placed in the footnote area at the bottom of the page.
  4. Footnotes are numbered sequentially using CSS counters.
  5. The footnote area is styled according to the @footnote rules defined in the stylesheet.

The implementation handles edge cases such as footnotes that span multiple pages, footnotes in marginalia, and footnotes within tables or other complex layouts.Handling footnotes in complex layouts requires careful tracking of which page each footnote belongs to, especially when content flows across page boundaries. PaperView uses the break token system to manage this.

Styling Footnotes

Footnotes can be styled using the @footnote at-rule within @page rules:

@page {
    @footnote {
        border-top: 1px solid #333;
        padding-top: 6pt;
        margin-top: 12pt;
        font-size: 9pt;
    }
}

Additionally, the ::footnote-call and ::footnote-marker pseudo-elements can be styled to customize the appearance of the reference markers:The pseudo-element syntax follows standard CSS conventions. For example, ::footnote-call { color: blue; } would make all footnote reference numbers blue.

::footnote-call {
    color: #0066cc;
    font-size: 0.75em;
}

::footnote-marker {
    font-weight: bold;
    color: #333;
}

Accessibility Considerations

Footnotes present unique accessibility challenges. Screen readers need to be able to navigate between footnote references and their content. PaperView's implementation includes click handlers that allow users to jump from a footnote reference to its content in the footnote area.Future versions may include ARIA attributes to improve screen reader navigation, such as aria-describedby linking references to their footnotes.

Historical Context

Footnotes have a long history in typography, dating back to medieval manuscripts. The modern footnote convention of sequential numbering at the bottom of the page was standardized in the 18th century. CSS Paged Media brings this traditional typographic feature to the web, enabling high-quality digital publications.The history of footnotes is fascinating. They evolved from marginal glosses in medieval manuscripts, where scribes would add explanatory notes in the margins. As printing developed, these marginal notes moved to the bottom of the page, becoming the footnotes we know today.
