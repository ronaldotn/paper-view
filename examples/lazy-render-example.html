Introduction to Lazy Rendering

Lazy rendering is a performance optimization technique that only renders pages when they enter the viewport. This is especially useful for large documents with 100+ pages, where rendering all pages at once would cause significant memory usage and slow initial load times.

With lazy rendering, the document is chunked in the background to determine page breaks, but only the pages visible in the viewport (plus a configurable buffer) are actually rendered in the DOM. Pages outside the viewport are shown as lightweight placeholders.

Eager Rendering (Traditional)

  • All pages rendered at once
  • High memory usage
  • Slow initial load for large docs
  • Browser may freeze during render

Lazy Rendering (New)

  • Only visible pages rendered
  • Low memory footprint
  • Fast initial load
  • Smooth scrolling experience

Usage

// Enable lazy rendering
const previewer = new PaperView.Previewer({
    viewMode: "single",
    lazyRender: true,
    lazyRenderOptions: {
        bufferPages: 3,        // Pages to render around viewport
        unloadPages: 5,        // Pages away before unloading
        enableUnload: true,    // Unload pages when out of view
        onPageRender: (index, data) => {
            console.log(`Page ${index + 1} rendered`);
        },
        onPageUnload: (index, data) => {
            console.log(`Page ${index + 1} unloaded`);
        }
    }
});

// Preview with lazy rendering
const flow = await previewer.preview(content, renderTo, stylesheets);

// Get render info
const info = previewer.getLazyRenderInfo();
console.log(info);
// { totalPages: 150, renderedPages: 12, progress: 8, chunkingComplete: true }

// Scroll to specific page
previewer.scrollToPage(50);  // By index
previewer.scrollToPageNumber(51);  // By page number

// Events
previewer.on("chunkingComplete", (data) => {
    console.log(`Chunked ${data.totalPages} pages`);
});

previewer.on("lazyPageRendered", (data) => {
    console.log(`Page ${data.index + 1} rendered`);
});

previewer.on("lazyPageUnloaded", (data) => {
    console.log(`Page ${data.index + 1} unloaded`);
});

How It Works

  1. Background Chunking: The document is chunked in a hidden container to determine all page breaks.
  2. Placeholder Creation: Lightweight placeholders are created for all pages in the visible container.
  3. IntersectionObserver: Monitors which pages enter/leave the viewport.
  4. On-Demand Rendering: Pages are rendered only when they enter the viewport (plus buffer).
  5. Memory Management: Pages can be unloaded when they move far from the viewport.
