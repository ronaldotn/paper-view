1. Introduction to Paged Media

Paged media refers to content that is divided into discrete pages, as opposed to continuous scrolling media. This includes printed documents, PDFs, and other paginated formats. The CSS Paged Media Module provides powerful tools for controlling page layout, margins, and content generation.

1.1 What is Paged Media?

Paged media is fundamentally different from continuous media. In paged media, content is broken into pages, each with its own dimensions, margins, and potentially different layouts. This requires special CSS properties and rules that don't apply to screen media.

The concept of paged media has been part of CSS since CSS2, but the modern CSS Paged Media Module Level 3 extends these capabilities significantly, adding support for page floats, footnotes, leaders, and more sophisticated page layout controls.

1.2 CSS Paged Media Module

The CSS Paged Media Module defines several key concepts:

@page rule - Defines the page box, including size, margins, and margin boxes for headers and footers.

Page selectors - Allow different styles for first, left, right, or named pages.

Page breaks - Control where content breaks from one page to the next using break-before, break-after, and break-inside properties.

1.3 Page Boxes and Margins

Each page in paged media consists of a page box containing a page area and margin boxes. The page area holds the document content, while the 16 margin boxes (four corners and three on each side) can contain generated content like page numbers, headers, and running titles.

2. Generated Content

Generated content is content that is not present in the source document but is created by CSS. This is essential for paged media, where you need to add page numbers, running headers, table of contents entries, and other elements that depend on the document structure.

2.1 The content Property

The content property is used with pseudo-elements (::before and ::after) to insert generated content. In paged media, it supports special functions like target-counter(), target-text(), and leader().

2.2 The leader() Function

The leader() function creates a leader line - a repeating pattern that fills the space between two elements. This is most commonly used in tables of contents to create dotted lines between chapter titles and page numbers.

Note: This entire page demonstrates the leader() function in action. The Table of Contents on the first page uses leader(dotted) to create the dotted lines between titles and page numbers.

The function accepts several values:

leader(dotted) - Creates a dotted leader using ". " (dot followed by space). This is the most common choice for tables of contents.

leader(solid) - Creates a solid line leader, useful for a more formal appearance.

leader(space) - Creates a leader with wider spacing between dots.

leader(dots) - Creates a leader using bullet characters.

leader("char") - Creates a leader with a custom character, such as leader("-") or leader("·").

2.3 target-counter() Function

The target-counter() function retrieves the value of a counter at the element referenced by a URL. This is essential for creating tables of contents, where you need to display the page number of each chapter.

Example usage in a TOC entry:

.toc-entry::after { content: leader(dotted) target-counter(attr(href url), page); }

2.4 target-text() Function

The target-text() function retrieves the text content of the element referenced by a URL. This can be used to create cross-references that automatically update when the document changes.

3. Page Floats

Page floats allow figures, tables, and other block elements to be positioned at specific areas of a page, such as the top, bottom, or occupying the entire page. This provides much more control than traditional CSS floats.

3.1 Float: Top and Bottom

Elements with float: top are positioned at the top of the page, before the main content flow. Elements with float: bottom are positioned at the bottom. These are ideal for figures and tables that should not interrupt the reading flow.

3.2 Float: Page

Elements with float: page occupy the entire page, centered and prominent. This is perfect for full-page figures, charts, or important visual content that needs maximum impact.

3.3 Float-Defer and Clear

The float-defer property allows you to defer the placement of a float by a specified number of pages. The clear property with values like page-top ensures that floats start on a new page after clearing previous floats.

4. Footnotes

Footnotes are annotations that appear at the bottom of a page, referenced by a superscript number in the main text. The CSS Paged Media Module provides comprehensive support for creating and styling footnotes.

4.1 Float: Footnote

Elements with float: footnote are extracted from the text flow and placed in a footnote area at the bottom of the page. A footnote call (superscript number) is inserted at the original position.

4.2 Footnote Display and Policy

The footnote-display property controls how the footnote appears (block, inline, or compact). The footnote-policy property determines how footnotes are handled when they don't fit on the current page.

5. Hyphenation

Hyphenation controls how words are broken at line endings. Proper hyphenation is essential for professional typesetting, especially in narrow columns or justified text.

5.1 Hyphens Property

The hyphens property accepts three values: auto (automatic hyphenation based on language), manual (only at explicit hyphenation points), and none (no hyphenation).

5.2 Hyphenate-Limit-Chars

The hyphenate-limit-chars property controls the minimum number of characters before and after a hyphenation point, as well as the minimum total word length for hyphenation.

6. Advanced Layout Techniques

Beyond the basic features, CSS Paged Media offers advanced techniques for creating sophisticated document layouts.

6.1 Running Headers and Footers

Running headers and footers use the position: running() property to define content that is then displayed in page margin boxes using element(). This allows chapter titles, author names, and other information to appear automatically in headers.

6.2 String Sets

String sets allow you to capture text from the document and use it in generated content. This is useful for creating running headers that show the current chapter or section title.

6.3 Named Pages

Named pages allow you to assign specific page types to elements, enabling different layouts for different sections of a document. For example, a title page might have no headers or page numbers, while content pages have full headers.

7. Best Practices and Tips

Creating professional paged media documents requires attention to detail and understanding of both CSS and typography principles.

7.1 Performance Optimization

Large documents can be slow to paginate. Use techniques like lazy rendering, pagination caching, and progressive loading to improve performance. Avoid overly complex CSS selectors and minimize the use of expensive properties.

7.2 Cross-Browser Compatibility

Not all browsers support CSS Paged Media features equally. Use a polyfill like PaperView (paged.js) to ensure consistent rendering across browsers. Test your documents in multiple environments.

7.3 Debugging Techniques

Use browser developer tools to inspect generated content, page boxes, and margin areas. Add temporary borders to page elements to visualize layout. Check the console for warnings about missing targets or invalid selectors.

8. Conclusion

CSS Paged Media provides a powerful set of tools for creating professional-quality printed documents from web content. The leader() function, demonstrated throughout this document in the Table of Contents, is just one of many features that enable sophisticated page layout.

By combining leaders with target counters, page floats, footnotes, hyphenation, and generated content, you can create documents that rival those produced by traditional desktop publishing software.

The key to success is understanding how these features work together and testing your documents thoroughly across different output formats and devices.
